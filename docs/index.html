Introduction

Single Sign-On (SSO) allows users to authenticate once and gain access to multiple applications without logging in separately to each one.

Getting Started

Ensure you have the necessary permissions and access to the identity provider you plan to use.

Configuration Steps

  1. Choose an Identity Provider: Select an identity provider that supports SSO.
  2. Register Your Application: Register your app in Microsoft Entra ID.
  3. Configure Redirect URIs: Add your GitHub Pages URL as the redirect URI.
  4. Implement Authentication Logic: Add MSAL auth in your web app.
